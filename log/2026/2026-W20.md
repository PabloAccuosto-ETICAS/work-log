## Wiselook audit

- **Partner API auth diagnostic + email to Roberto (Sat May 09).**
  Verified keys in `.env` against Roberto's email — match character
  by character. Discriminative test of auth headers via curl: the
  server processes `Authorization: Bearer` and rejects other formats
  with a clear message. Same call via Swagger UI returns HTTP 500
  instead of 401, with the same key. The `-i` flag on curl revealed
  both responses include `x-clerk-auth-*` headers indicating Clerk
  middleware processes the partner API key as a JWT and fails
  (expected — keys are not JWTs); behaviour after Clerk diverges
  by request origin (clean 401 via curl, crash via browser). Email
  sent to Roberto with curl reproducer + full response (CCs include
  Borja Gracia, new contact). Reply pending. Filter probes promoted
  to actionable-now while waiting.
